Python 核心知识

1. 解释 Python 的 GIL (全局解释器锁)

GIL 是 CPython 解释器中的一个互斥锁，它保证了在同一进程中，只有一个线程能同时执行 Python 的字节码。它的存在主要是为了简化 CPython 的内存管理。对于 CPU 密集型任务，应使用多进程；对于 I/O 密集型任务，多线程仍然可以提升效率，因为线程在等待 I/O 时会释放 GIL。

2. Python 的内存管理机制是怎样的？

主要由三部分组成：引用计数（主要机制，当对象引用计数为0时回收）、标记-清除（解决循环引用问题）和分代回收（对标记-清除的优化，提高效率）。

3. 什么是装饰器 (Decorator)？请写一个例子。

装饰器是一个函数，它接受另一个函数作为参数，并返回一个新的函数，用于在不修改原函数代码的情况下增加额外功能。

import time

def timing_decorator(func):
    def wrapper(*args, **kwargs):
        start_time = time.time()
        result = func(*args, **kwargs)
        end_time = time.time()
        print(f"'{func.__name__}' ran in: {end_time - start_time:.4f} sec")
        return result
    return wrapper

@timing_decorator
def my_function(n):
    sum = 0
    for i in range(n):
        sum += i
    return sum

my_function(1000000)
# 输出: 'my_function' ran in: 0.0489 sec

4. 什么是生成器 (Generator) 和迭代器 (Iterator)？

迭代器是实现了 `__iter__()` 和 `__next__()` 方法的对象。生成器是一种特殊的迭代器，使用 `yield` 关键字返回值，它支持惰性求值，非常节省内存。

# 生成器函数
def fibonacci_generator(n):
    a, b = 0, 1
    count = 0
    while count < n:
        yield a
        a, b = b, a + b
        count += 1

# 使用
for num in fibonacci_generator(5):
    print(num)

数据结构与算法

1. 如何在 Python 中反转一个字符串？

最简洁的方法是使用切片 `[::-1]`。

my_string = "hello"
reversed_string = my_string[::-1]
print(reversed_string) # 输出: "olleh"

2. 给定一个列表，移除所有重复的元素并保持顺序。

可以利用 `dict.fromkeys()`，因为在 Python 3.7+ 中字典是保持插入顺序的。

my_list = [1, 2, 2, 3, 4, 3, 5]
unique_list_ordered = list(dict.fromkeys(my_list))
print(unique_list_ordered) # 输出: [1, 2, 3, 4, 5]

3. 实现一个二分查找算法。

二分查找适用于已排序的序列，通过不断将搜索区间减半来查找目标值。

def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1 # 未找到

Web 框架 (Django/Flask)

1. 解释 Django 的 MVT 架构。

Django 遵循 MVT (Model-View-Template) 架构。Model 负责数据层和数据库交互；View 负责业务逻辑处理，接收请求并返回响应；Template 负责表现层，生成用户看到的 HTML 页面。

2. Flask 和 Django 的主要区别是什么？

Django 是一个“自带电池”的全功能框架，提供了 ORM、Admin 后台等大量内置组件，适合大型复杂项目。Flask 是一个微框架，核心简单，灵活性极高，需要开发者自己选择组件，适合小型应用和 API 开发。

数据库与系统设计

1. 什么是数据库索引？为什么它能提高查询速度？

索引是一种特殊的数据结构（如 B-Tree），它存储了表中特定列的值和其物理地址。当查询时，数据库可以利用索引快速定位数据行，避免全表扫描，从而大幅提升查询速度，其原理类似于书的目录。

2. 如何设计一个短链接服务？

核心思路是为每个长链接生成一个唯一的短字符串（短码），并存储它们的映射关系。生成短码：可以使用自增 ID 转换为62进制的方法来保证唯一且简短。存储：使用数据库（如 MySQL 或 Redis）存储 `short_key` 和 `long_url` 的映射。重定向：当用户访问短链接时，服务器根据 `short_key` 查找 `long_url`，并返回 301/302 重定向响应。
